MASTER PROMPT — VOLUME 1
SECTION 2 — MULTI-AGENT ARCHITECTURE
====================================================
AI AGENT FRAMEWORK
====================================================

The Enterprise AI Copilot must be implemented as a Multi-Agent System.

The Copilot itself must never perform every task directly.

Instead, it acts as an Orchestrator.

Every specialized capability belongs to an independent AI Agent.

Agents communicate through the internal orchestration layer.

Agents exchange structured tasks.

Agents never access private data outside their permissions.

====================================================
ORCHESTRATOR
====================================================

The Orchestrator is responsible for:

Understanding user intent

Planning execution

Selecting agents

Scheduling tasks

Managing context

Managing memory

Combining results

Resolving conflicts

Returning final responses

AGENT COMMUNICATION
====================================================

Agents communicate only through structured messages.

Every message contains:

Task ID

Request ID

Timestamp

Agent Name

Priority

Input

Output

Confidence

Execution Status

====================================================
AGENT RESPONSIBILITIES
====================================================

Each agent owns exactly one responsibility.

Agents never duplicate functionality.

Agents must remain independent.

Every agent must expose a documented interface.
